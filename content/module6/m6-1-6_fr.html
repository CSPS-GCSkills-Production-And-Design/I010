Contrôle des versions

GCDOCS vous permet également de sélectionner le type de contrôle des versions qui sera appliqué aux documents. Il existe deux types de contrôle :

Contrôle standard des versions : ce type de contrôle applique le format linéaire aux versions, ce qui signifie que la version 1 sera suivie de la version 2, laquelle sera suivie de la version 3, etc.

Contrôle avancé des versions : ce type de contrôle détermine les versions principales et les versions secondaires. Les versions principales sont représentées par 1.0, 2.0, 3.0, etc., et les versions secondaires, par 0.1, 0.2, 0.3, etc.

L’avantage du contrôle avancé des versions, c’est que certaines versions deviennent des « versions principales ». Comme certains utilisateurs ont l’autorisation de voir le contenu des versions principales seulement, il s’agit d’un excellent moyen de gérer l’accès à certaines versions.
